## 核心规程 (Core Rule)
本工作流运行 `Retro` 阶段。目标是:将本轮暴露的流程缺陷、规则漏洞与知识盲区沉淀为长期优化。

## 共享知识库公约 (Shared Knowledge Location)
- 7 个工作流共用一套单层跨项目共享知识库:`C:\knowledge`。
- `C:\knowledge` 只保存跨项目长期知识(协作规则、编码偏好、设计偏好、长期经验、流程偏好)。
- 项目结构、模块边界、接口契约、业务规则、项目技术决策必须维护在当前项目目录内(如 `docs/architecture/`),不写入 `C:\knowledge`。
- 禁止在 `C:\knowledge` 中创建 `global/` / `project/` / `global_workflows/` 目录或 core/stages 等冗余副本。

## 必读规程 (Required Reading Procedure — 固定 7 步)
> 本节定义本入口必须触发的物理读取动作。门禁细则与禁止行为见 `core/stage-gates.md` 与 `stages/6-retro.md`,本入口不重抄。

1. **加载规则**:物理读取本入口列出的"必须加载"中所有 `core/` / `references/` / `stages/` / `templates/` 文件,登记到 Evidence Ledger。
2. **回读上游产物**:物理读取本轮已有的所有阶段产物文件(Requirement / Plan / Task / Review),登记 `E-id`;任一缺失即先打回补齐。
3. **验收输入**:在产物 `Upstream Consumption` 章节逐条标注与本轮问题相关的所有 `AC-id` / `M-id` / `T-id` / `F-id` / `Constraints`(`Consumed / Not Applicable / Deferred`)。
4. **探索源码、图谱、知识**:按 `stages/6-retro.md` 中查询层默认 `Skipped` 规则处理 codegraph(初始化层不豁免);物理读取 `C:\knowledge\INDEX.md` 与 `docs/architecture/INDEX.md`(若涉及结构沉淀);若复盘涉及具体代码缺陷复核,可调用 `codegraph.find_callers / find_definition` 并登记 `E-id`。
5. **产出文件**:按 `templates/retro-report.md` 创建或更新 `.workflow/retro-report.md`,Workflow / Rule Changes 必须挂 `F-id` 并写明落点文件;实际更新至少 1 条 `C:\knowledge/*` 跨项目知识或 `docs/architecture/*` 项目架构文档,记录到 Knowledge Update Evidence。
6. **回读自检**:执行 `core/anti-hallucination.md` 中的 Verification Order;核对 Claim Evidence Map 中所有完成断言挂真实 `E-id`;若没有实际规则/知识更新,本阶段不得关闭。
7. **下一步建议**:若发现能力缺口推荐 `/7-技能巡检`;若问题已系统级解决则本阶段闭合;否则只写 `Blockers` + `Rollback Advice`。
