## 核心规程 (Core Rule)
本工作流运行 `Requirement` 阶段。目标是:将模糊需求收敛为可执行、可验证、可交接的 `Requirement Contract`。

## 共享知识库公约 (Shared Knowledge Location)
- 7 个工作流共用一套单层跨项目共享知识库:`C:\knowledge`。
- `C:\knowledge` 只保存跨项目长期知识(协作规则、编码偏好、设计偏好、长期经验、流程偏好)。
- 项目结构、模块边界、接口契约、业务规则、项目技术决策必须维护在当前项目目录内(如 `docs/architecture/`),不写入 `C:\knowledge`。
- 禁止在 `C:\knowledge` 中创建 `global/` / `project/` / `global_workflows/` 目录或 core/stages 等冗余副本。

## 必读规程 (Required Reading Procedure — 固定 7 步)
> 本节定义本入口必须触发的物理读取动作。门禁细则与禁止行为见 `core/stage-gates.md` 与 `stages/1-requirement.md`,本入口不重抄。

1. **加载规则**:物理读取本入口列出的"必须加载"中所有 `core/` / `references/` / `stages/` / `templates/` 文件,登记到 Evidence Ledger。
2. **回读上游产物**:本阶段允许从空上下文开始,写 `Not Applicable; new round`;若存在更高阶段历史产物,登记其读取证据并写明本次为重新启动。
3. **验收输入**:在产物 `Upstream Consumption` 章节标注外部输入(用户原始需求、相关 issue、设计稿)状态;Reference Rate 必须 100%。
4. **探索源码、图谱、知识**:按 `stages/1-requirement.md` 与 `core/codegraph-engine.md` 中本阶段最少动作清单执行 codegraph 检查与查询;物理读取 `C:\knowledge\INDEX.md` 与 `docs/architecture/INDEX.md`(若存在),登记 `E-id`。
5. **产出文件**:按 `templates/requirement-contract.md` 创建或更新 `.workflow/requirement-contract.md`,Acceptance Criteria 必须分配 `AC-id`。
6. **回读自检**:执行 `core/anti-hallucination.md` 中的 Verification Order;核对 Claim Evidence Map 中每条断言的 `Supporting E-ids` 是否真实存在;核对 Upstream Reference Rate=100%。
7. **下一步建议**:全部门禁通过则唯一下一步 `/2-方案设计`;否则只写 `Blockers` + `Rollback Advice`。

## 阻断违规与门禁细则
本入口的硬门禁、禁止行为、伪证据判定与回退条件统一由以下文件承载,**禁止在本入口重抄**:
- `core/stage-gates.md`:7 步执行序列、Input / Scope / Evidence / Upstream Consumption / CodeGraph / Exit / Rollback Gate。
- `core/anti-hallucination.md`:伪证据黑名单、Claim Evidence Map、工具失败语义、Sequential Thinking 回包字段要求。
- `core/codegraph-engine.md`:初始化层强制规则、本阶段最少动作清单、跳过证据格式、不可用回退路径。
- `core/evidence-engine.md`:证据台账 6 列结构、高置信结论规则。
- `stages/1-requirement.md`:本阶段 Allowed / Disallowed Actions、ID 约定、Required Output、Handoff Checks、Rollback Conditions。
