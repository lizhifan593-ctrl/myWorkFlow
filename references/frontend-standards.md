# 前端架构与体验基准 (Frontend Standards)

## 1. 架构原则与隔离机制
- **视图与逻辑解耦 (Smart vs Dumb)**：页面容器组件（Smart）负责数据流调度、状态分发，基础组件（Dumb）仅通过 props 接收数据并触发 emit，禁止直接修改内部状态或与外部环境产生副作用。
- **状态抽离 (Composables / Hooks)**：复杂业务逻辑和接口请求严禁直接堆砌在视图组件内，必须剥离到独立的复用单元中。
- **去魔法字符串 (No Magic Strings)**：所有枚举、状态字典表禁止直接在 HTML 模板硬编码，必须抽离为常量集中管理，并使用结构化方式渲染。

## 2. 视觉设计与交互防御
- **设计风格 (Linear 极简风与毛玻璃)**：以深色模式或干净底色为主，使用 1px 细线、高数据密度排版，大幅克制原生阴影。
- **视觉防御三法则**：
  - **防跳跃 (Anti-jump)**：涉及异步高度变化的面板，必须前置预留 `min-height` 或固定高度，禁止加载完后暴力推开页面引起重排。
  - **防闪白 (Anti-flash)**：深色主题下加载态必须显式置空背景（如 `background="transparent"`），禁止使用默认的白色半透明遮罩。
  - **防挤压 (Anti-squeeze)**：浮层/抽屉等组件必须使用绝对定位 (`absolute` / `fixed`)，严禁侵入普通文档流挤占正常布局空间。
  - **防穿透与浮层失焦 (Teleport Control)**：在实现带有 `click-outside` 自动收起边界的自定义悬浮面板时，若内部混用 Element Plus 的弹出型组件（如 `el-select`、`el-dialog`），**必须**为这些子组件显式附加 `:teleported="false"`。否则这些子菜单会被默认传送渲染至 HTML `body` 底部，导致点击菜单时被父级判定为“点击面板外部”，引发面板异常自毁。
- **优雅过渡**：所有的 `:hover` 与状态切换必须配备 `0.2s - 0.3s ease` 缓动；涉及悬浮放大的组件推荐利用 `opacity` 或是外层光晕实现，在包含 `overflow-y: auto` 的定宽容器中，严禁直接使用 `transform: scale()` 放大，避免触发极丑陋的物理切割（Clip）或排版抖动。

## 3. 网络与生命周期容错
- **单页应用长连接生存期**：WebSocket / SSE 的通信管道需挂载在 Auth（登录与登出）行为的生命周期上，登入主动连，登出主动断，不可随单页路由随意装卸。
- **网络异常容错 (无感自愈)**：当后端由于发布重启或异常出现 502/504 等 50x 错误时，前端全局守卫严禁直接清除 token 强制踢出。应保持界面静默重试或优雅降级，仅在真正的 401/403 授权拦截时才进行退出。
- **数据流安全**：
  - 前端组件的 Props 在接收可能异步赋予的 `null` 时必须做好类型兼容或给予默认值拦截，防止警告或阻塞下游渲染。
  - 大数据长列表或二进制流下载（Blob），必须在拦截器或响应器中严谨判断数据类型，避免把二进制流强制 toString 渲染为 `[object Object]` 乱码。

## 4. 语法与工程构建规范
- **防御性 Import 严控**：严禁为了在开发期强行绕过某些 IDE 错误检查（如语法报错）而使用畸形的语法（例如 `import { ... } from 'import { ... }'`）。每次依赖注入变更后，必须脑内/自我评估代码合法性，避免在 Vite 编译打包时引发阻断性崩溃（如 `Identifier has already been declared` 或致命的编译失败）。
