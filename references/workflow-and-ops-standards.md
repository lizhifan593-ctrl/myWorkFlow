# 工作流交互与部署运维规范 (Workflow & Ops Standards)

## 1. AI 助手的“举证与质疑”法则 (Challenge & Prove Protocol)
- **主动举证原则 (Prove before Code)**：在执行架构规划和核心编码前，AI 不能仅作为唯命是从的“打字机”。AI 必须向开发者清晰地阐述这套实现方案背后的深层原因，利用数据流向、框架源码逻辑或历史踩坑案例来**证明**该方案的高可用性、防御性与合理性。
- **主动质疑原则 (Challenge the Request)**：当发现用户提出的需求指令中存在明显的“过度工程化倾向”、“逻辑前后矛盾”或严重偏离框架最佳实践基准时，AI **必须主动踩下刹车**。通过反问的形式向开发者点明系统级风险或逻辑盲区，并提供平替的、更优雅或更稳健的重构选项供用户定夺，严禁盲目执行错误指令。

## 2. 运维与环境安全规范
- **持久化容器密码陷阱**：在使用 Docker 等容器化部署带持久卷配置的数据中心（如 MySQL）时，通常环境参数（如 `MYSQL_ROOT_PASSWORD`）只在卷的首次初始化时生效。后续如遇容器连接拒绝或密码变更，直接修改环境文件 `.env` 极大概率无效，排查时需优先怀疑历史凭证，必要时须采用挂载特殊启动参量（如 `skip-grant-tables`）强制修正。
- **网关代理与长连接打字机效应防阻塞**：在使用 Nginx / API Gateway 等网关代理 SSE (Server-Sent Events) 单向长连接（如大模型流式输出或消息推送）时，**必须**在代理配置域下显式关停缓冲行为（如加上 `proxy_buffering off;`），否则网关的默认数据块缓冲机制会导致客户端的流式打字机效果出现严重迟滞、卡顿甚至请求阻断。
- **挂载环境的无感权限隔离**：任何由程序驱动、在服务端产生落盘操作（例如临时上传点 `uploads/`、缓存或日志持久化目录），在服务初始启动点或编排文件挂载时，必须强制确认宿主机对应的系统执行用户对该类文件夹有完整、不可被剥夺的**读写权限**，防范低级而致命的权限拒绝抛错。
- **Docker 静态防御网络（网段冲突隔离）**：在编写 `docker-compose.yml` 部署应用时，**严禁**依赖 Docker 默认分配的桥接网络子网。必须在底部 `networks` 域显式声明不常被使用的偏僻静态子网（如 `10.254.0.0/16`）。防止 Docker 自动生成的 `192.168.x.x` 或 `172.x.x.x` 等网段与宿主机物理局域网网段重叠，造成主机路由表紊乱、局域网数据包被吸入黑洞，从而引发极其严重的服务器“断网失联”（SSH 中断）事故。

## 3. 全栈编码与联调防御法则 (Full-stack Coding & Integration Defense Protocol)
- **API 载荷边界防御**：在构建向后端发送请求的 Payload 时，必须**显式交叉核对**后端 Pydantic/DTO 模型的类型声明（尤其是 `int` 与 `str`）。如前端常用的带前缀 ID（如 `g_2`、`u_1`）绝不可直接向后端投递，必须在发起 HTTP 请求前的中枢控制层完成前缀剥除和字段重构。
- **状态机的闭环销毁机制**：当引入新的全局事件（如 SSE 广播的数据销毁事件 `group_disbanded`）时，**绝不能只增加事件本身**。必须立刻全局检索前端所有的状态监听层，确保该事件被截获并触发界面的**强制重排**（如强制清理 `activePartnerId`），坚决杜绝“幽灵页面”驻留引发的空指针错乱。
- **Vue 组件上下文防御**：在 Vue 3 `<script setup>` 中抽离复用函数时，切勿凭直觉盲写。涉及外部传入的变量，**必须强关联 `props.` 访问**（如 `props.currentUserId`）；涉及内部响应式状态的，**必须强关联 `.value`**。必须在提交代码前通过全局搜素复核一遍变量的作用域，防止 `ReferenceError` 导致渲染引擎挂载崩溃。
- **UI 框架隐式样式覆盖防御**：在使用第三方 UI 库（如 Element Plus）的组件堆叠时（如多个 `<el-button>`），必须警惕框架默认注入的相邻组件间距（如 `margin-left`）。对于高危的、全宽 `100%` 或垂直排列的按钮组，必须主动加上防御性的重置样式（如 `margin-left: 0 !important;`）进行视觉压制，防范不可预期的版面错位。
