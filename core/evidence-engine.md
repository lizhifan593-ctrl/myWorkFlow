# Evidence Engine

## Purpose
统一定义什么可以作为结论依据，以及证据如何进入产物和回退说明，防止模型凭感觉宣称完成或正确。

## Allowed Evidence Sources
- 文件读取结果
- 搜索结果
- 命令执行结果
- 用户明确回复
- 已写入并可回读的产物内容

## Disallowed Substitutes
以下内容不能替代证据：
- 仅凭记忆的断言
- 仅凭“看起来合理”的判断
- 仅凭实现者主观印象的自证
- 未回读的已修改内容

## Evidence Format
建议统一写成：

```markdown
## Evidence Notes
- Source:
- Location:
- Conclusion:
```

## When Evidence Is Required
以下场景必须附证据：
- 文件、接口、模块、规则是否存在
- 当前产物是否满足上游要求
- 当前任务或改动是否已落地
- 当前任务是否已形成最小验证闭环
- 当前阶段是否可以进入下一阶段
- 是否需要回退，以及回退依据是什么
- 某条知识是否仍然有效
- 任何“无问题、已完成闭环、已验证通过、可交付、可上线”的结论

## High-Confidence Claim Guidance
- “已完成”至少应有对应改动已落地的证据。
- “已验证通过”至少应有直接验证结果作为证据。
- “已完成闭环”至少应同时具备产物更新证据与验证证据。
- “可交付”或“可上线”至少应明确写出支撑该判断的验证范围；若验证范围不足，不得提前给出该类结论。

## Result-Focused Guidance
- 证据的目标是支撑结果判断，不要求所有模型都用同一种证明路径。
- 可以根据任务类型选择最合适的证据来源，只要结论可被核验。
- 若存在多个可行证据，优先选择最直接、最省歧义的那一个。

## Evidence Failure Handling
- 若证据与结论不一致，优先修正结论。
- 若当前拿不到足够证据，不得提前宣称完成，应继续检查或显式阻塞。
