# Prototype Structure

## Purpose
- 这个结构原型页要解释什么问题？
- 若涉及 UI 决策，说明它支撑了哪一类结构讨论，而不是直接替代讨论。

## Target Screens / Components
- 涉及哪些页面或组件？

## Layout Blocks
- 页面有哪些主要区块？
- 每个区块承担什么作用？

## Interaction Paths
- 用户从哪里进入？
- 有哪些关键操作路径？

## State Variants
- 需要表达哪些状态？例如空态、加载态、错误态、成功态、展开态、编辑态。

## Exclusions
- 本原型明确不包含哪些内容？例如视觉设计、品牌样式、真实数据。

## Mandatory Reads
- 支撑该原型的需求契约、实施方案、相关页面或架构入口。

## Read Evidence
- 实际读取了哪些文件或原有页面结构。
- 这些读取如何影响了区块划分、交互路径或状态设计。

## Structure Decision Notes
- 本原型支撑了哪些结构讨论。
- 哪些 UI 决策已经被讨论过，哪些仍然开放。

## Blockers
- 当前原型仍缺哪些输入或约束。
- 若无，写明 `None`。
