## 核心规程 (Core Rule)
本工作流运行 `Review` 阶段。目标是:基于增量改动检查实现是否满足需求、方案与任务,并识别是否存在更简单、更稳的实现方式。

## 共享知识库公约 (Shared Knowledge Location)
- 7 个工作流共用一套单层跨项目共享知识库:`C:\knowledge`。
- `C:\knowledge` 只保存跨项目长期知识(协作规则、编码偏好、设计偏好、长期经验、流程偏好)。
- 项目结构、模块边界、接口契约、业务规则、项目技术决策必须维护在当前项目目录内(如 `docs/architecture/`),不写入 `C:\knowledge`。
- 禁止在 `C:\knowledge` 中创建 `global/` / `project/` / `global_workflows/` 目录或 core/stages 等冗余副本。

## 必读规程 (Required Reading Procedure — 固定 7 步)
> 本节定义本入口必须触发的物理读取动作。门禁细则与禁止行为见 `core/stage-gates.md` 与 `stages/5-review.md`,本入口不重抄。

1. **加载规则**:物理读取本入口列出的"必须加载"中所有 `core/` / `references/` / `stages/` / `templates/` 文件,登记到 Evidence Ledger。
2. **回读上游产物**:物理读取 `.workflow/requirement-contract.md`、`.workflow/implementation-plan.md`、`.workflow/task-list.md` 与本次增量改动 / Diff,登记 `E-id`;任一缺失即触发 Rollback。
3. **验收输入**:在产物 `Upstream Consumption` 章节逐条标注上游 `AC-id` / `M-id` / `T-id` / `Constraints`(`Consumed / Not Applicable / Deferred`),Reference Rate 必须 100%。
4. **探索源码、图谱、知识**:按 `stages/5-review.md` 与 `core/codegraph-engine.md` 中本阶段最少动作清单执行 codegraph 查询(对修改过的所有核心符号执行 `find_callers` 做回归面盘点);物理读取 `C:\knowledge\INDEX.md` 与 `docs/architecture/INDEX.md`(若审查涉及结构一致性),登记 `E-id`。
5. **产出文件**:按 `templates/review-report.md` 创建或更新 `.workflow/review-report.md`,Findings 必须分配 `F-id`,三段 Alignment 表(Requirement / Plan / Task)按 `AC-id / M-id / T-id` 逐条对账。
6. **回读自检**:执行 `core/anti-hallucination.md` 中的 Verification Order;Final Assessment 写 `Pass` 时所有完成断言必须挂高置信 `E-id`;Findings 写 `None` 必须给出"未发现问题"的检索证据。
7. **下一步建议**:全部门禁通过则唯一下一步 `/6-复盘优化`;偏差严重时按严重度回退到对应阶段;否则只写 `Blockers` + `Rollback Advice`。

## 阻断违规与门禁细则
本入口的硬门禁、禁止行为、伪证据判定与回退条件统一由以下文件承载,**禁止在本入口重抄**:
- `core/stage-gates.md`:7 步执行序列、Input / Scope / Evidence / Upstream Consumption / CodeGraph / Exit / Rollback Gate。
- `core/anti-hallucination.md`:伪证据黑名单、Claim Evidence Map、工具失败语义、Sequential Thinking 回包字段要求。
- `core/codegraph-engine.md`:初始化层强制规则、本阶段最少动作清单、跳过证据格式、不可用回退路径。
- `core/evidence-engine.md`:证据台账 6 列结构、高置信结论规则。
- `stages/5-review.md`:本阶段 Allowed / Disallowed Actions、ID 约定、Required Output、Handoff Checks、Rollback Conditions。

## Required Output Fields (Reference)
> 阶段产物 `.workflow/<artifact>.md` **必须**包含下述字段;字段顺序与定义见 `templates/review-report.md`,语义见 `core/evidence-engine.md` 与 `core/anti-hallucination.md`。

- Upstream Acceptance / Upstream Consumption(覆盖上游所有结构化项,Reference Rate=100%)

- Knowledge Applied
- 阶段专有内容(详见模板)
- Open Issues / Blockers / Rollback Advice

- Knowledge Update Evidence
- Claim Evidence Map(每条完成断言挂真实 E-id;无证据只能写 Blocked / Not verified / Pending)
