# 工作流 3: 编码实现

- `core/hermes-harness.md`
- `core/hermes-cognition.md`

## 核心规程 (Core Rule)
本工作流运行 `Implementation` 阶段。目标是作为精密执行器，按照物理 `Task List` 精确改动代码，对每个任务进行局部验证并留下 Command 证据，更新任务清单状态，最后生成 `.workflow/walkthrough.md` 物理产物。

你必须严格执行 `core/hermes-harness.md` 中规定的 **Hermes 5 步循环** 规程：
1. **加载与物理回读**：读取上述文件，加载任务清单。
2. **输入与门禁验收**：逐项对账待办任务，确定物理修改的目标范围。
3. **源码与知识探索**：在对物理文件写入前，必须执行 CodeGraph 依赖 callers 探勘。
4. **产出落盘与字段校验**：按 `templates/walkthrough.md` 创建落盘走查文档，并将 `task-list.md` 标记更新。
5. **证据映射与自检**：运行单元测试或编译以收集高置信 `E-id` 证据，挂载自检后输出唯一步骤 `/4-审查与验证`。
